Important Information Regarding Report Title, Description, SQL Query, and Report Header

Do not use double quotes in your Report Title. Doing so will cause the report to not output correctly because the double quotes will cause html issues on the report output page. Buttons will not display and row colors will be off.

If your Report Description starts with a <, such as <br>, the description will not export. The same is true for Directions and Notes. Either avoid using an < at the start or if you do use one, after you do the export, copy the report description manually into the export file. Do the same for Directions and Notes if needed.

Avoid using the following in the select clause in the SQL Query: double quotes around field aliases (other than externalexpression, aliases are not needed as column names will be added in the Report Header area instead), and reserved characters, such as colon and semi-colon. For a colon use chr(58) and for a semi-colon use chr(59). If you're concatenating and need a comma, use chr(44) instead. Also, you can drag the bottom left-hand corner of the input box down to have a bigger box.

To display the external expression in a report (the external expression is sometimes different than the value in the expression field), add externalexpression as an alias to the expression field in your sql. For example, if your sql has cc.expression as one of the fields, use cc.expression externalexpression in the sql. The externalexpression alias will trigger coding in the report to display the external expression instead.

In the Report Header, make sure you have an adequate number of headings, otherwise your report output will be off. Do not press enter after any report header - just keep them all on a continuous line. Pressing enter after any of them will cause white space to appear above the report. Buttons will not display and row colors will be off. With Report Headings, there are five class= commands you can use in a <th>: DCID, DCID DISPLAY, HIDE, NOCF, and ROWGRP. The DCID ones are explained under the Student Selection tab. The HIDE command is useful to hide a column needed for the SQL but you don't want to display, or when using the show/hide columns button and you want to hide columns initially that the user can show using the button. The NOCF command is used when Column Filtering is enabled to remove filtering for a visible column. The ROWGRP command is used when Row Grouping is enabled and uses the column for row grouping purposes.

Important Information Regarding Field Level Security

Starting with version 4.5, you can add field level security to your reports. If a person has No Access rights to a field, you can include the field in the report and *'s will appear instead of the actual value on the output. For it to happen you must have coding in the Field Level Security box, which follows the format of mapfield=column#>table.fieldname. If there's more than one column, add a comma, and then another column#>table.fieldname. The column# will be the column number in the Report Header box, and table.fieldname will be replaced by actual table and field names.

For example, if you have the DOB field from the students table in a report, and in the Report Header box that is column 2, then it would look like mapfield=2>students.dob. If you also have a reference to the SSN field in the report and that's in column 5, then the full command would be mapfield=2>students.dob,5>students.ssn.

Clicking on the Build Map Field will create a generic mapfield command which you use as a guide. You'll need to replace the table.fieldname parts with the actual table and field names (do not use aliases - use the actual names for both), and remove any part of the coding that you do not need.

Student Selection

Student Selection will allow you to make the students listed the current selection. There are two ways to do it and you can use either one or both.
Option A: The selection is based on the original sql query. A link will appear at the bottom of the results page labeled 'Original Query Current Selection', and clicking on it will return all the students returned by the query below, even if the list has been searched and filtered down on the results page.
Use Option A
Student Selection Query (Leave blank if you're not using Option A.)
Build Query
Option B: The selection is based on the students on the results page. A button will appear on the page labeled 'Make Current Selection', and when clicked, those students on the screen at the time will become the current selection.
To use, you must include the student DCID field in the SELECT clause of your statement and add either:
  • class="DCID" to the th coding of the column heading of the column with the student DCID - for example, <th class="DCID">Student DCID</th>, this will hide the DCID column from view and export, or
  • class="DCID DISPLAY" to the th coding of the column heading for the column with the student DCID - for example, <th class="DCID DISPLAY">Student DCID</th>, this will display the DCID column on the screen and show in exports

Important: Row Grouping will add erroneous information to the dcid values and make the selection incorrect. If you have Row Grouping enabled on the Results tab, then Option B will be ignored.

Important Information Regarding Column Filtering and Row Grouping

Column Filtering and Row Grouping do not interact well with Show/Hide Columns or Column Reordering. If you enable either Column Filtering or Row Grouping, then Show/Hide Columns and Column Reordering will be disabled on the output. Column Filtering needs the search box, so the search box will appear even if it hasn't been checked above.

Also, Row Grouping causes issues with Option B Student Selection, which is the one that adds the Make Current Selection button. It will add erroneous information to the dcid values and make the selection incorrect. If you have Row Grouping enabled, then Option B will be ignored and no Make Current Selection button will appear on the output.
